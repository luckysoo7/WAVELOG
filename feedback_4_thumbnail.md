# 썸네일 기능 기획 — bcamp-daily

## 현재 상태

`page.tsx` (홈) 및 `date/[date]/page.tsx` 양쪽 모두 동일한 트랙 리스트 구조를 사용:

```
<li class="track-row flex items-center gap-4 py-3.5 group">
  [번호]   [제목 + 아티스트]   [▶ 링크 (hover 시 표시)]
```

- 각 row는 번호(w-6) + 텍스트 블록(flex-1) + 링크 아이콘(w-6) 3-column 레이아웃
- `videoId` 있을 때만 ▶ 아이콘이 hover 시 나타남 (opacity 0 → 1)
- 썸네일 URL: `https://img.youtube.com/vi/{videoId}/mqdefault.jpg` (320×180)
- `matched: false` 곡은 videoId가 null — 썸네일 없음

현재 UI는 텍스트 전용. 시각적 밀도가 낮고, 라디오 선곡표라는 감성이 잘 드러나지 않음.

## 옵션 A: Parallax Vinyl — 스크롤 연동 앨범 사이드패널

**어떻게 보이는가:**

트랙 리스트 오른편 고정 패널(`position: sticky`, top: 30%)에 현재 스크롤 위치 기준
"포커스 트랙"의 썸네일이 대형(약 200×200, 정사각형 크롭)으로 떠 있음.
썸네일 아래에 트랙 번호와 제목이 작게 표시됨.
스크롤 시 포커스 트랙이 바뀌면서 썸네일이 페이드 크로스로 전환됨.
`videoId`가 없는 트랙에 포커스가 오면 패널은 희미한 노이즈 텍스처로 대체됨 (곡 있음/없음 시각화).

데스크톱에서만 동작. 모바일(< 768px)에서는 패널 숨김, row에 작은 점(·) 표시로 대체.

레이아웃 변경: 현재 `max-w-[760px]`를 `max-w-[1000px]`로 확장, 우측 240px를 사이드패널로 사용.

**UX:**

스크롤이 곧 "음악 감상"의 은유. 읽으면서 자연스럽게 각 곡을 시각적으로 접하게 됨.
인터랙션 없이 수동적으로 감상하는 라디오 청취 경험과 잘 맞음.

**구현 난이도:** 높음

- `IntersectionObserver`로 각 row의 뷰포트 진입 감지 → 포커스 트랙 state 관리
- CSS `crossfade` 또는 absolute overlay + opacity 트랜지션
- `'use client'` 컴포넌트 필요 (현재 서버 컴포넌트)
- 레이아웃 max-width 변경 → 반응형 중단점 재조정 필요

**장점:**
- 인터랙션 없이도 자동으로 작동 → 낮은 진입장벽
- 긴 선곡표(20~30곡)에서도 시각적 피로 없이 정보 전달
- "라디오 방송 시퀀스를 따라간다"는 내러티브 감각

**단점:**
- 모바일에서 사실상 비활성 — 모바일 사용자에게는 기능이 없는 것과 동일
- 레이아웃 구조 변경(max-width 확장)이 기존 헤더 타이포그래피 리듬에 영향
- `videoId` 없는 곡이 많을 때 패널이 자주 공백 상태가 됨

## 옵션 B: Spectral Row — 썸네일 색상 추출로 row 자체를 물들이기

**어떻게 보이는가:**

각 트랙 row의 배경에 해당 곡의 YouTube 썸네일에서 추출한 **지배색(dominant color)**을
매우 낮은 불투명도(약 6~10%)로 깔아 전체 row를 부드럽게 물들임.
썸네일 자체는 보이지 않음. 색깔만 남아 있는 형태.

hover 시 동일 색상이 20~25%로 강해지고, 왼쪽 세로 3px 컬러 바(accent line)가 나타남.

`videoId` 없는 곡은 배경 없음 + 살짝 낮은 opacity로 시각적으로 구분됨.

색상 추출은 빌드 타임에 수행 → `songs.yaml` 또는 별도 JSON에 `dominantColor: "#a83b2c"` 저장.
런타임 추출(client-side canvas) 방식은 성능·SSR 호환 문제로 배제.

**UX:**

"보이지 않지만 느껴지는" 미학. 각 곡마다 미묘하게 다른 온도감.
클래식 곡은 차갑고 어두운 계열, 팝은 밝은 계열 등 장르가 색깔로 암시됨 (의도치 않은 부산물이지만 흥미로움).
시각적 변화가 너무 크지 않아 텍스트 가독성 유지.

**구현 난이도:** 중간

- `color-thief` 라이브러리 또는 `sharp` + 직접 구현으로 빌드 타임 색상 추출
- 크롤러 파이프라인에 색상 추출 단계 추가 (Python `colorthief` 또는 `Pillow`)
- CSS custom property로 row에 `--row-color` 주입 → 인라인 스타일
- 클라이언트 코드 변경 없음 (서버 컴포넌트 유지)

**장점:**
- 썸네일 UI 중 서버 컴포넌트를 유지할 수 있는 유일한 옵션
- 모바일/데스크톱 동일하게 작동
- 레이아웃 변경 없음
- 가장 독창적 — "썸네일을 보여주는" 접근을 완전히 해체함

**단점:**
- 빌드 파이프라인에 색상 추출 단계 추가 필요 (크롤러 수정)
- 추출한 색상이 항상 "예쁘게" 나오지 않을 수 있음 (보정 로직 필요)
- 색상 데이터 없으면 기존 UI와 동일 — 초기 데이터 소급 적용 필요

## 옵션 C: Reveal on Tap — 트랙 클릭 시 씨네마틱 확장 패널

**어떻게 보이는가:**

기본 상태: 현재 UI와 동일 (번호 + 텍스트 + ▶ 아이콘).

`videoId`가 있는 트랙을 클릭하면 해당 row가 아래로 확장되며 두 영역이 나타남:
- 왼쪽: 16:9 썸네일 (전체 컨테이너 너비의 약 45%). 클릭 시 YouTube로 이동.
- 오른쪽: 제목(대형), 아티스트, "YouTube에서 듣기" CTA 버튼.
  썸네일 아래에 블러 처리된 동일 이미지가 배경으로 깔려 씨네마틱한 depth 효과.

확장 높이: 약 120px. `max-height` + CSS transition으로 애니메이션.
다른 row 클릭 시 기존 확장 닫힘 (accordion 패턴).

**UX:**

능동적 탐색 경험. 궁금한 곡만 골라서 확인하는 흐름.
라디오 선곡표 맥락에서 "이 곡이 뭐지?" 하는 순간의 욕구를 정확히 해소함.
YouTube 연결 CTA가 크게 보여 클릭 전환율 향상 기대.

**구현 난이도:** 중간

- `'use client'` 컴포넌트로 TrackList 분리 필요
- `expandedId` state + `max-height` transition
- 서버 컴포넌트(page.tsx)는 유지, TrackList만 클라이언트화
- 모바일에서도 동일하게 작동 (탭 기반이므로 오히려 모바일 친화적)

**장점:**
- 썸네일을 "볼 이유가 있는 순간"에만 보여줌 → 정보 밀도 균형
- 모바일에서도 완전히 작동
- 레이아웃 구조 변경 없음
- 현재 ▶ 아이콘 UX와 자연스럽게 연결됨

**단점:**
- 인터랙션이 필요하므로 수동적 감상 흐름을 깨뜨림
- Accordion은 흔한 패턴 — 독창성 면에서 옵션 A, B보다 낮음
- `videoId` 없는 곡은 클릭해도 반응 없음 → 사용자 혼란 가능 (비활성 표시 필요)

## 추천

추천: **옵션 B (Spectral Row)** — 단, 크롤러 수정 비용을 감수할 의향이 있을 때.

이유: 세 옵션 중 가장 독창적이고 bcamp-daily의 "라디오 감성" 정체성과 가장 잘 맞음.
썸네일을 "보여주는" 대신 "느끼게" 하는 접근은 디자인 originality 기준에서 확실히 차별화됨.
레이아웃 변경도 없고 서버 컴포넌트를 유지할 수 있어 기술적 debt도 가장 적음.

단, 크롤러 수정 없이 빠르게 가시적 결과를 보고 싶다면 **옵션 C**가 차선.
옵션 C는 구현 범위가 명확하고, 모바일까지 커버되며, 기존 ▶ 아이콘 UX를 자연스럽게 발전시킴.

**옵션 A는 비추천.** 데스크톱 전용인 점, 레이아웃 구조 변경 필요, 모바일 사용자 소외가 현재 단계에서 부담이 큼.
