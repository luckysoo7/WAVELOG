# ADR-002: SSG 전략 — 빌드 타임 DB 읽기

날짜: 2026-04-12
상태: 확정

## 배경

Next.js 웹 서버 운영 방식 결정 필요. 선택지:
- SSR (서버사이드 렌더링) — 런타임에 DB 쿼리
- SSG (정적 사이트 생성) — 빌드 타임에 DB 읽어서 HTML 생성

## 결정

**SSG + Vercel.** 빌드 타임에 SQLite를 읽어 정적 HTML 생성.

## 이유

- 런타임 서버 불필요 → 비용 $0 (Vercel 무료 플랜)
- 크롤러 실행(GitHub Actions 22:00) → git commit → Vercel 자동 재빌드 → 새 에피소드 반영
- 에피소드 데이터는 하루 1번만 갱신 → SSR 불필요
- SQLite를 git에 포함 → Vercel 빌드 환경에서 DB 파일 접근 가능

## 한계

- 빌드 시간 = 에피소드 수 비례로 증가 (현재 약 37페이지, 수 초)
- 실시간 데이터 불가 — 크롤러 실행 후 Vercel 빌드 완료까지 수 분 지연
- DB 파일이 git에 포함되므로 개인정보/민감 데이터 저장 금지

## 결과

크롤러 → DB 갱신 → git push → Vercel 자동 빌드 → 배포 완료 파이프라인.
